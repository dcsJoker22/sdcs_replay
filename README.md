# SDCS Campaign Replay Viewer

Interactive web-based replay of [Strategic DCS](https://strategic-dcs.com) campaign sessions. Displays aircraft tracks, ground units, base ownership, kill events, objective progress, and pilot telemetry on a live map.

## Live Demo

Hosted at [strategic-dcs.com](https://strategic-dcs.com) — currently in development.

---

## Stack

| Component | Technology |
|-----------|-----------|
| Parser | Python 3 — converts Tacview `.acmi` files to session JSON |
| Viewer | Single-file HTML + Leaflet.js — no build step |
| Data | Pre-processed JSON files, one per campaign session |
| Hosting | Static file server (any) |

## Setup

### Requirements

```bash
pip install watchdog   # for watch_acmi.py only — no other dependencies
```

### 1. Parse all campaigns (batch)

```powershell
# Run from project root — parses everything in raw/ and writes to public/data/
python parse_acmi.py
```

Or parse a single session:

```powershell
python parse_acmi.py "raw\2026-02-26 Germany\20260226_074617.zip.acmi"
```

Both `.acmi` and `.zip.acmi` input formats are handled. Output goes to:
`public/data/<campaign_folder>/session_<stem>.json`

### 2. Build the campaign index

```powershell
python build_campaigns.py
```

Scans all `public/data/*/session_*.json` and writes `public/campaigns.json`.
Campaign names and theatre IDs are auto-detected from folder names.

### 3. Serve locally

```powershell
python -m http.server 8080 --directory public/
```

Open `http://localhost:8080/sdcs_replay_v1.52.html`.

## Adding a New Campaign

1. Create `raw/YYYY-MM-DD CampaignName/` (include map name in folder, e.g. `2026-04-01 Germany`)
2. Drop `.acmi` or `.zip.acmi` files in
3. `python parse_acmi.py` (batch) or parse individual files
4. `python build_campaigns.py`

Folder name auto-detection:

| Folder contains | Theatre | campaign_id |
|-----------------|---------|-------------|
| `Caucasus` | Caucasus | 190 |
| `Germany` | Germany | 192 |
| `Syria` | Syria | 193 |
| `Gulf` or `Persian` | Persian Gulf | 189 |

```
        ▼

        │

  build_campaigns.py
        │
        ▼

        │
        ▼

```

## Viewer Features

- Campaign and session selectors (newest session pre-selected)
- 9 type-specific SVG aircraft silhouettes, heading rotation, smooth interpolation
- Altitude and Mach telemetry labels
- Air-launched weapon tracks (missiles, bombs, rockets)
- Kill events with explosion animations and PvP highlighting in kill feed
- Ground unit diamonds with fade/hide on `Visible=0` (despawn/retreat)
- Base and objective ownership tracking via Shelter3 events
- Objective progress bars (blue vs red)
- Player detail panel with flight history and kill log
- Filters: player aircraft, AI aircraft, ground units, ground labels, trails, kills, bases, weapons
- UI scale slider for icon/label size
- FLOT (front line overlay) — code present, currently disabled

## Versioning

HTML files are versioned (`sdcs_replay_v1.52.html`). Never overwrite an existing versioned file — always increment to the next version number. `campaigns.json` and session JSON files are regenerated freely.
